# Principles

## Build for first-time local AI users

The site should help smart beginners make good hardware decisions without needing to already understand CUDA, ROCm, quantization, VRAM limits, or server-card jank.

## Practical value beats raw benchmarks

The best GPU is not always the fastest or cheapest. Rankings should consider price, tokens/s, VRAM, software maturity, noise, power, setup pain, and used-buying risk.

## Be opinionated, but transparent

Give clear verdicts. Mark uncertainty. Link sources. Explain scoring. Avoid fake precision.

## Trust first, monetization second

Affiliate links are fine, but they must not influence rankings. The site should feel like a useful tool, not an SEO farm.

## Local AI is about ownership

The project supports the idea that users should be able to own useful AI capability locally instead of depending entirely on cloud tools.

## Avoid beginner traps

Cheap server cards, old datacenter GPUs, and experimental stacks can look good on paper. The site should clearly flag when something is only suitable for tinkerers.

## Keep the MVP simple

One hero model. One useful table. Short docs. Static-first architecture. Add complexity only after the core table is useful.

## Prefer clear human explanations

Use plain language, short verdicts, and practical tradeoffs. Do not bury users in benchmark jargon.

## Data should be humble

Benchmarks depend on model, quant, backend, context, drivers, and settings. Treat numbers as practical buying estimates, not universal truth.
